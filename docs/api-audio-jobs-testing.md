# Audio Job API — Local integration test

## Summary

I ran a local integration smoke test for the newly added Audio Job API endpoints in `apps/api`.

## What I did

- Created a local SQLite database using the `apps/api/.env` file (DATABASE_URL="file:./prisma/dev.db").
- Generated the Prisma client and applied the schema (migrations/seed run).
- Started the API server in dev mode (port 4000).
- Executed the included demo script `scripts/demo-audio-api.sh` which exercises the endpoints:
  - POST `/api/audio/jobs` (create jobs)
  - GET `/api/audio/jobs` (list jobs, pagination)
  - GET `/api/audio/jobs/:id` (fetch job details)
  - POST validation failure case (non-audio contentType)

## Results

- The API server started successfully at `http://localhost:4000`.
- Two jobs were created (separate & enhance) and returned 201 payloads with `QUEUED` status.
- Listings and fetching by ID returned the expected shapes and pagination info.
- Validation error returned 400 with field-level detail when supplying `contentType: "application/pdf"`.

## Notes

- Jobs are persisted to `apps/api/prisma/dev.db` (SQLite). The code currently does not enqueue jobs for processing (status stays `QUEUED`).
- The demo script printed one jq parse warning due to how output is echoed, but the test flow completed and validated the endpoints.
