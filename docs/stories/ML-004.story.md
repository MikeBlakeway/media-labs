## Details

### Problem

Currently, generated video outputs (MP4) are not playable inline in the UI. Instead they appear as a broken/unfound image with no controls to preview them. Users must download the MP4 and open it in an external application to view results, which is inconvenient and breaks the expected interactive workflow.

### Impact

- Poor user experience: previewing outputs requires extra steps outside the app.
- Increased friction for iterative workflows and debugging.
- Some users may assume the video generation failed when they see a broken image placeholder.

### Scope

In scope:

- Implement an in-app video player component capable of playing MP4 outputs served via signed URLs.
- Provide playback controls: play/pause, seek/skip forward and back, time display (current/total), volume control, and fullscreen toggle.
- Support progressive loading and error states (show spinner while loading, retry button on failure).
- Ensure player works with signed URL expiry by fetching a fresh signed URL from the server API if playback fails due to authorization.
- Integrate the player into `src/components/MediaDisplay.tsx` and `src/components/ResultHistory.tsx` where video outputs are shown.
- Ensure accessibility: keyboard controls, focusable controls, and proper ARIA roles/labels.

Out of scope:

- Advanced editing features (trimming, frame-by-frame scrubbing) — these can be follow-ups.

---

## Acceptance criteria

- Given a generated MP4 output, when the user views the result in the UI, then the video renders inline in a player with visible controls (play/pause, seek, time, volume, fullscreen).
- Given the signed URL expires, when the client tries to play the video, then the player or API retries and fetches a refreshed signed URL automatically and resumes playback where possible.
- Given the video is loading, when buffering occurs, then a loading indicator is shown and playback remains responsive.
- Given an error occurs (corrupt file, permission denied), when the player fails to load, then a clear error message and a Download button are displayed.
- Given keyboard navigation, when the player has focus, then user can use Space to toggle play/pause, arrow keys to seek small steps, and F to toggle fullscreen (standard media key patterns).
- Component tests and a simple integration test for signed URL refresh are included.

## Notes & Implementation Hints

- Use the HTML5 `<video>` element with a custom control bar or a lightweight React player wrapper.
- Prefer native controls for mobile and fallback gracefully to custom controls for desktops.
- To handle signed URL expiry, attempt to fetch the video resource; if a 403/401 occurs, call the server endpoint to request a renewed signed URL and update the `src` of the `<video>` element.
- Consider using `MediaSource` API only if progressive streaming or chunked playback is required; prefer direct `src` swapping for simplicity.
- For testability, abstract the signed-url refresh into a helper module that can be unit tested.
