# MMI-006: ControlNet Integration

## Details

### Problem

The multi-modal inference worker needs ControlNet capabilities to provide guided image generation using edge detection (Canny) and depth estimation. ControlNet enables precise control over image generation by providing structural guidance from input images.

### Impact

Without ControlNet integration:

- Users cannot control image composition using reference images
- Guided image generation workflows will be unavailable
- The system lacks precision control capabilities that differentiate it from basic text-to-image
- Advanced creative workflows requiring structural control cannot be implemented
- Integration with existing Media Labs ControlNet templates will be impossible

### Scope

This story covers implementing ControlNet handlers for Canny edge detection and depth estimation, integrating with the FLUX.1 base model established in MMI-005, and providing controlled image generation capabilities.

---

## Acceptance criteria

- Given a control image and prompt, when ControlNet inference is requested with Canny mode, then edge-guided image generation completes within 20 seconds.
- Given a control image and prompt, when ControlNet inference is requested with depth mode, then depth-guided image generation produces structurally accurate results.
- Given invalid control images, when preprocessing fails, then appropriate error messages explain the image requirements and constraints.
- Given control strength parameters, when provided, then the generated image reflects the specified level of structural guidance.
- Given memory constraints, when ControlNet models are loaded, then they share base model components efficiently with FLUX.1.
- All ControlNet operations must integrate with the established model management and routing systems.
