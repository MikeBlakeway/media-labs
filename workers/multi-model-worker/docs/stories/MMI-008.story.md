## Details

### Problem

The multi-modal inference worker needs direct text-to-video generation capabilities using LTX-Video 2B distilled model. This enables users to create video content directly from text descriptions without requiring an initial image, completing the core video generation capabilities of the system.

### Impact

Without LTX-Video text-to-video functionality:

- Users cannot generate videos directly from text prompts
- The system lacks comprehensive video creation capabilities
- Text-to-video workflows essential for content creators will be missing
- The multi-modal promise of supporting all major inference types cannot be fulfilled
- Competitive positioning against other video generation platforms will be weakened

### Scope

This story covers implementing the LTX-Video handler for text-to-video generation, including the 2B distilled model integration, video sequence generation, and optimization for the target inference time of under 45 seconds.

---

## Acceptance criteria

- Given a text prompt, when text-to-video inference is requested, then LTX-Video generates coherent video content within 45 seconds.
- Given video parameters (duration, fps, resolution), when specified, then the generated video matches the requested specifications.
- Given complex text descriptions, when processed, then the generated video accurately reflects the described scene, action, and style.
- Given memory constraints, when LTX-Video loads, then it integrates with model management without causing OOM errors.
- Given concurrent requests, when multiple text-to-video jobs run, then each completes successfully with appropriate resource allocation.
- All generated videos must maintain temporal consistency and visual quality suitable for Media Labs video workflows.
